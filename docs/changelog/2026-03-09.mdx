## Project config (arc.toml)

Arc now supports project-level configuration via `arc.toml`. Previously, you had to pass full file paths to `arc run` and configure each run individually. Now, you can define workflow aliases, pull request defaults, and retro settings at the project level. Arc discovers `arc.toml` by walking up from the current directory, similar to how Cargo or Node find their config files.

```toml title="arc.toml"
[workflows]
implement = "arc/workflows/implement.toml"
hello = "arc/workflows/hello.toml"

[retro]
enabled = false
```

```bash
arc run implement  # resolves via arc.toml
```

## exe.dev sandbox maturation

The exe.dev sandbox provider now supports the full workflow lifecycle. Previously, exe.dev VMs could run agent stages but lacked git integration, cancellation, and checkpointing — making them unusable for real multi-stage workflows. Now, Arc automatically clones the local git repo into exe.dev VMs, supports remote git checkpointing between stages, and wires up cancellation tokens so stopping a run cleanly tears down the VM.

You can also specify a custom container image and connect via SSH:

```toml title="run.toml"
[execution]
environment = "exe"

[execution.exe]
image = "my-custom-image:latest"
```

```bash
arc run --ssh my-workflow.dot
```

## `arc cp` — copy files to and from sandboxes

After a workflow run completes, you often need to retrieve files from the sandbox — build artifacts, generated reports, or logs. Previously, there was no easy way to get files out of a completed run's sandbox. Now, `arc cp` lets you copy files in either direction using the run ID.

```bash
arc cp <run-id>:/path/in/sandbox ./local-dir   # download
arc cp ./local-file <run-id>:/path/in/sandbox   # upload
```

## `arc pr create` — create PRs from completed runs

Sometimes you want to review a run's changes before opening a PR, or a run finishes without auto-PR configured. `arc pr create` lets you create a pull request after the fact for any completed workflow run, using the persisted manifest, conclusion, and diff from the run's log directory. PRs are created as drafts by default.

```bash
arc pr create <run-id>
```

## `arc system df` — disk usage reporting

Run logs and worktrees accumulate over time. `arc system df` shows a breakdown of disk usage across your run logs directory and any preserved worktrees, so you know when it's time to run `arc system prune`.

```bash
arc system df
```
